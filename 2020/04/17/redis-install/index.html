Redis 笔记（二）：Redis 安装与使用

发表于 2020-04-17 分类于 Tech ， IT ， Database 阅读次数： 评论数：
本文字数： 1.1k 阅读时长 ≈ 1 分钟

在 Vbox 下安装了 Ubuntu Server 后，继续安装 redis，并通过端口映射到主机，使其可以被外部访问（在 Goland 中写代码可以通过 redis 相关的库直接连接、调用）

安装

  1. 通过 apt 包安装：

    sudo apt-get update
sudo apt-get install redis-server

  2. 通过源码安装：

# 以下步骤如果报错提示 permission denied，请使用 root 权限。
# “make test” 不是必需的步骤，如果嫌麻烦可以直接从第四步跳到第七步，也就是在 redis 文件夹中执行 make 后，直接 make install。

1. wget http://download.redis.io/releases/redis-5.0.8.tar.gz

2. tar -zxvf redis-5.0.8.tar.gz

3. cd redis-5.0.8/

4. make     # 在编译完成后，输出信息的最后一行为 "Hint: It's a good idea to run 'make test' ;)"
            # 此时如果输入 "make test" 会报错 "You need tcl 8.5 or newer in order to run the Redis test"
            # 正确做法是先安装最新版的 tcl

    4.1 cd ..

    4.2 wget https://prdownloads.sourceforge.net/tcl/tcl8.6.10-src.tar.gz

    4.3 tar zvxf tcl8.6.10-src.tar.gz

    4.4 cd tcl8.6.10/unix

    4.5 ./configure

    4.6 make

    4.7 make install

5. cd ../redis-5.0.8/

6. make test

7. make install

Redis 命令说明

自带二进制相关命令介绍

redis-benchmark     # 性能测试工具
redis-check-aof     # 用于修复对应格式文件的命令
redis-check-rdb     # 同上
redis-cli           # 启动 redis 客户端
redis-sentinel      # 哨兵模式
redis-server        # 启动 redis 服务

启动 Redis 服务

redis-server redis.conf     # 如不填写 "redis.conf" 则使用默认配置文件
                            # 这里建议单独建立一个文件夹，copy 一份 redis.conf 过去
                            # 用 copy 的配置文件启动，这样可以保证根目录的配置文件完整性

# 上述启动方式为前台启动，并不方便。此时可以修改配置文件改为后台启动。

vim redis.conf
    daemonize no -> yes     # 以守护进程的形式运行
